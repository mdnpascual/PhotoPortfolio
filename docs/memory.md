## Resolved Decisions

### Tech Stack
- **Framework:** React + Vite + TypeScript (strict, `noUnusedLocals: true`)
- **Styling:** Tailwind CSS v4 вЂ” `@import "tailwindcss"` in index.css, `@tailwindcss/postcss` plugin
- **tsconfig:** `"types": ["vite/client"]` added вЂ” required for `import.meta.env` typing
- **Animations:** Framer Motion (transitions, UI) + GSAP + ScrollTrigger (scroll-driven, REQUIRED)
- **Smooth Scroll:** `lenis` (darkroomengineering package)
- **Routing:** React Router DOM В· browser history В· `basename="/PhotoPortfolio"` В· fade 300ms via Framer Motion AnimatePresence
- **Image Comparison:** `react-compare-slider`
- **Carousel:** `embla-carousel-react`
- **Hosting:** GitHub Pages В· `base: "/PhotoPortfolio/"` in vite.config.ts
- **Node.js:** v24.1.0 locally В· `node-version: 24` in GitHub Actions

### Image Strategy
- Orientation-based variants (portrait vs landscape), NOT device-based
- Detection: `viewport.height > viewport.width` = portrait, else landscape
- Naming: `{section}-{name}-{aspect}-{orientation}.webp`
- All WebP pre-generated by user вЂ” no build-time conversion
- Loading state: Tailwind `animate-spin` spinner В· Error state: yellow triangle exclamation SVG

### Content & Data
- User authors content by editing JSON files directly (no CMS)
- Data validation: TypeScript compile-time only (no runtime library)
- JSON schema structure defined in `feature-11.md`

### Gallery Categories
- **Birds В· Wildlife В· Landscape В· People** (Flowers removed)

### Deployment
- `base: "/PhotoPortfolio/"` вЂ” vite.config.ts
- `basename="/PhotoPortfolio"` вЂ” BrowserRouter in main.tsx
- `public/404.html` stores original URL in `sessionStorage.redirect` then redirects to repo root
- `index.html` restore script reads `sessionStorage.redirect` and calls `history.replaceState` (fixed in F07)
- GitHub Actions: push to `main` в†’ build в†’ deploy to `gh-pages` branch
- No environment variables or API keys needed
